Type of project: SaaS
Role: Main developer

Upscale Forms is a SaaS product dedicated to create highly customizable forms with a capacity to intagrate with other applications.

The challenge

We had 4 main goals on this project. Have a interface where the user could place and crontrol fields. Have the option to style the form as the user chooses. Have the possibility to integrate with any outside application or systems. Have an easy way to display the form in the web or mobile.

1: The builder

For the whole client we choose to use Next.Js with typescript to add the benefits of type safety. The wole interface consists of many screens, however the most complex one is the screen that allows users to place fields as they please, manage fields properties and create layout rules. We created a grid systems composed by rows and columns, with that fields could be placed almost anywhere it needed to be placed. To make the experince more dynamic and symple we decided to implement a drag and drop systems, at first we used an amazing npm package called react-beautiful-dnd but as our demand grew we had to develop our drag and drop logic from scratch

2: The styler

To achive a high customization capability we decided to implement a space for the user to type their own css. We created the components page, a place where the user could create css styles for certain field type, once the component was created it could be used multiple times to style multiple fields. Also we design a marketplace to share you custom design with other users. To make the css typing a similar experince as typing it on your own IDE we used Monaco.

3: The Commander

The interactivity module is separated in 2 main sections. The actions section with are resposible for inetractions while the user is filling the form, the user can select triggers and create custom functions to perform any action it might need like display messagens or fill certain fields. The other section is called Workflow with is responsible for inetgration tasks after the user has submitted the form. We have a flow chart where it can be created in the choosen order tasks that will be executed. This tasks could be anything from custom functions to easy to setup Oauth integrations to other applications. The custom functions used in both sections are TypeScript functions written inside of the platform, that hava a type safe system with every field on the form created on the builder and also a log an error list to check any outcome or be aware of malfunctions in your code.

4: Share it

At the currently moment we have 3 ways to share a created form. The embed options is an Iframe that can be placed in any website. The React.Js module that allows for easy integration in any react application developed, it can be found in the NPM regestry. Also we have the options to display your form on a mobile app with out React-Native package that is also available in teh NPM regestry. In the future we plan to create packages for other frameworks like Vue.Js and a SDK for vanilla js.
